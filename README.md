<summary><b>🔷 SaaSMetrics — Multi-Tenant SaaS Analytics Dashboard</b></summary>
<br/>

Architected and deployed a full-stack, multi-tenant SaaS analytics platform with **server-enforced tenant data isolation**, verified through automated tests to guarantee organizations can never access each other's data — even under direct API manipulation. Implemented JWT authentication using short-lived access tokens with rotating httpOnly-cookie refresh tokens to mitigate XSS-based token theft, backed by a two-layer role-based access control system spanning backend middleware and frontend routing. <summary><b>🔷 Sonic Architecture — Concurrency-Safe Event Booking System</b></summary>
<br/>

Engineered a ticket booking backend that **guarantees zero double-bookings** under concurrent load, using atomic database transactions with conditional guards instead of fragile check-then-act logic. Validated correctness with a live load test firing **50 simultaneous booking requests** against the production deployment — confirming exactly one request succeeds while 49 are correctly rejected with proper conflict responses. Implemented automatically-expiring reservation holds via a background job and idempotent booking confirmation to safely absorb client retries without duplicate bookings.

| Category | Detail |
|---|---|
| **Stack** | React · TypeScript · Node.js · Express · PostgreSQL · Prisma · JWT · Concurrency Control |
| **Scale** | Production load-tested at 50 concurrent booking requests per resource |
| **Performance** | Atomic, conditional-guard transactions eliminating race conditions |
| **Security** | JWT-authenticated booking flow with idempotent confirmation handling |
| **Impact** | 100% double-booking prevention rate under verified concurrent load |
| **Repository** | [event-booking-concurrency-system.onrender.com](https://event-booking-concurrency-system.onrender.com/) |
